API Overview

The AUTOPAY API allows merchants to initiate M-Pesa STK Push payments, verify transaction status, receive webhook notifications and manage payment collections securely.

Quick Start

Complete these steps to start accepting payments with AUTOPAY.

  1. Create your AUTOPAY merchant account.
  2. Generate your API Keys.
  3. Configure your Webhook URL.
  4. Call the STK Push API.
  5. Receive payment confirmation through Webhook or Check Status.

Payment Collection Method

Every merchant configures a default collection method from their AUTOPAY dashboard. When an STK Push payment is successful, AUTOPAY automatically routes the payment to the configured destination.

Collection Method Payment Destination
AUTOPAY Wallet Funds are credited to your AUTOPAY Wallet balance.
M-PESA Till Funds are collected into your registered Till Number.
M-PESA Paybill Funds are collected into your registered Paybill account.
Bank Account Funds are collected into your verified Bank Account.
Your integration never needs to specify where payments should be sent. AUTOPAY automatically uses your configured Collection Method.

https://autopay.co.ke/api

How STK Push Works

When you initiate an STK Push request, AUTOPAY automatically identifies your merchant account from your API Key and checks your configured Collection Method. No additional destination parameters are required.

Step Description
1 Your server sends a POST request to /api/stk-push.
2 AUTOPAY validates your Secret API Key.
3 AUTOPAY identifies your merchant account.
4 AUTOPAY reads your configured Collection Method.
5 The customer receives the M-PESA STK Push prompt.
6 After successful payment, funds are automatically routed to your configured destination.
You never need to specify whether funds should go to your Wallet, Till, Paybill or Bank Account in the API request. AUTOPAY automatically routes the payment based on your merchant settings.

Request Parameters

Field Type Required Description
phone String Yes Customer M-PESA phone number in the format 2547XXXXXXXX.
amount Number Yes Payment amount in Kenyan Shillings (KES). Must be greater than 0.
balanceType String Yes Either wallet or service.

Automatic Payment Routing

Your application only initiates the payment request. After the customer completes the STK Push, AUTOPAY automatically routes the funds based on your configured Collection Method.

Configured Collection Method Payment Destination
AUTOPAY Wallet Merchant Wallet Balance
M-PESA Till Registered Till Number
M-PESA Paybill Registered Paybill Number
Bank Account Verified Bank Account
The API request does not require any destination field. AUTOPAY automatically determines where to collect the payment using your merchant account settings.

Best Practices

  • Validate phone numbers before sending requests.
  • Store checkoutRequestId in your database.
  • Always verify payment status.
  • Never expose Secret API Keys.
  • Handle duplicate webhook callbacks safely.
