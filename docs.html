API Overview

The AUTOPAY API allows merchants to initiate M-Pesa STK Push payments, verify transaction status, receive webhook notifications and manage payment collections securely.

Quick Start

Complete these steps to start accepting payments with AUTOPAY.

  1. Create your AUTOPAY merchant account.
  2. Generate your API Keys.
  3. Configure your Webhook URL.
  4. Call the STK Push API.
  5. Receive payment confirmation through Webhook or Check Status.

Base URL

https://autopay.co.ke/api

Available Endpoints

Method Endpoint Description
POST /api/stk-push Initiate STK Push
POST /api/check-status Check Transaction Status
POST /api/withdraw Withdraw Funds
POST /api/webhook Webhook Callback
GET /health Gateway Health

API Status

Current Environment: LIVE

Base URL: https://autopay.co.ke

API Version: v1

Authentication

Every request must include your Secret API Key. 

Authorization: Bearer sk_live_xxxxxxxxxxxxx

Authentication Flow

  1. Create your AUTOPAY merchant account.
  2. Generate your Live API Key.
  3. Include your Secret Key in the Authorization header.
  4. Use the API endpoints hosted on https://autopay.co.ke/api.
  5. Receive payment confirmation through Webhook or Check Status.

Required Headers

Authorization: Bearer YOUR_SECRET_KEY

Content-Type: application/json

Accept: application/json

API Tester

Use the Health endpoint to verify that the AUTOPAY API is online. 


GET https://autopay.co.ke/health

Payment Flow

1. Merchant sends an STK Push request.

2. Customer receives the M-Pesa payment prompt.

3. Customer enters their M-Pesa PIN.

4. AUTOPAY processes the payment.

5. Your webhook receives the payment notification.

6. Verify the payment using the Check Status endpoint if needed.
